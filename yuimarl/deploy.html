運用環境構築

Yuimarl の運用環境を、Amazonが提供するAmazon Web Servicesのクラウド環境である「EC2」の仮想サーバー上に構築する方法について記述します。ここでは、EC2上でのOSとして、Linuxディストリビューションの１つであるUbuntu を使用します。
ここで記述する例は、以下の環境とします。他の環境に構築する場合は、適宜、読み換えてください。

クラウド環境 Amazon EC2
OS Ubuntu
Java開発環境 JDK 1.8
Java EE 7 アプリケーションサーバー Payara
データベース MySQL
ビルドツール Apache Maven
SSH接続クライアント Tera Term

※ 以下の説明では、赤字の部分が入力する文字です。緑色の文字は説明で、入力しません。

  • Tera Termのインストール

Tera Termのインストール

Tera Termは、Windows用のターミナルエミュレータです。Amazon EC2のサーバーにSSH（Secure Shell）で接続するために使用します。Tera Term以外のターミナルエミュレータでも可能ですが、その場合は、RSA暗号によるSSH接続に対応したものをお使いください。

  1. 次のTera Termのダウンロードサイトにアクセスします。
    http://sourceforge.jp/projects/ttssh2/releases/
  2. ダウンロードの一覧から、最新版のexeファイルを選択してダウンロードします。（執筆時点では、「teraterm-4.91.exe」）
  3. ダウンロードしたファイルを実行して、Tera Termのインストールを行います。
    「コンポーネントの選択」では、初期表示のままで「次へ」を押してください。

Amazon EC2 仮想サーバーの構築

Amazon Web ServicesのEC2で、仮想サーバーを作る方法を記述します。

  1. Amazonのアカウントを持っていなければ、以下の手順で作成します。
    • Webブラウザで、Amazon Web Services のアドレスにアクセスします。
      https://aws.amazon.com/jp/
    • 「サインアップ」ボタンを押します。
    • 「Eメールまたは携帯番号:」を入力し、「私は新規ユーザーです。」をチェックし、「サインイン」ボタンを押します。
    • 「ログイン認証情報」画面で名前、メールアドレス、パスワードを入力します。
    • 「連絡先情報」画面で連絡先情報を入力します。
    • 「支払情報」画面で支払情報を入力します。
  2. Amazon Web Services のページで、「コンソールへサインイン」ボタンを押します。
    https://aws.amazon.com/jp/
  3. サインイン画面でメールアドレスとパスワードを入力してサインインします。
  4. AWSマネジメントコンソール画面で、「EC2」をクリックします。
  5. 「EC2ダッシュボード」画面で「インスタンスの作成」ボタンを押します。
  6. 「ステップ 1: Amazon マシンイメージ（AMI）」画面で「Ubuntu Server 14.04 LTS (HVM), SSD Volume Type」の「選択」ボタンを押します。
  7. 「ステップ 2: インスタンスタイプの選択」画面で、初期状態のまま、「確認と作成」ボタンを押します。
  8. 「ステップ 7: インスタンス作成の確認」画面で、「セキュリティグループの編集」をクリックします。
  9. 「ステップ 6: セキュリティグループの設定」画面で、「HTTP」、「HTTPS」、「カスタムTCPルール」（ポート範囲：8080）、「SSH」を追加します。（「送信元」は「任意の場所」を選択）

    「確認と作成」ボタンを押します。
  10. 「ステップ 7: インスタンス作成の確認」画面で、「作成」ボタンを押します。
  11. 「既存のキーペアを選択するか、新しいキーペアを作成します。」画面で、「新しいキーペアの作成」を選択し、「キーペア名」を入力します。キーペアのファイルをダウンロードしてから、「インスタンスの作成」ボタンを押します。
  12. 「作成ステータス」画面で、「インスタンスの表示」ボタンを押します。
  13. 「EC2ダッシュボード」画面で、表示された「パブリック IP」を確認してメモしておきます。

Tera TermからのSSH接続

  1. Tera Termを起動します。
  2. 「新しい接続」画面で、「ホスト」欄に、Amazon EC2のインスタンスのパブリックIPを入力します。
    「OK」ボタンを押します。
  3. 「セキュリティ警告」画面が表示されたら、「続行」ボタンを押します。
  4. 「SSH認証」画面で、「RSA/DSA/ECDSA/ED25519鍵を使う」をチェックし、「秘密鍵」ボタンを押して、上記でダウンロードしたキーペアのファイルを選択します。
    「ユーザ名」に「ubuntu」と入力し、「パスフレーズ」は空のままで、「OK」ボタンを押します。
  5. ログインが成功し、Amazon EC2のターミナル画面が表示されます。
  6. 次のコマンドを入力して、Ubuntuの更新を適用します。
    ubuntu@ip-172-31-23-251:~$sudo apt-get update
    （省略）
    Reading package lists... Done
    ubuntu@ip-172-31-23-251:~$sudo apt-get -y upgrade
    （省略）
    ubuntu@ip-172-31-23-251:~$
  7. タイムゾーンの設定
    • 次のコマンドを実行します。
      ubuntu@ip-172-31-23-251:~$sudo dpkg-reconfigure tzdata
    • 次の画面が表示されたら、リストから「Asia」を選択し、「Ok」ボタンを押します。
    • 次の画面が表示されたら、リストから「Tokyo」を選択し、「Ok」ボタンを押します。
    • dateコマンドを実行して、日本標準時(JST)で表示されることを確認します。
      ubuntu@ip-172-31-23-251:~$date
  8. 日本語環境の設定
    • 次のコマンドを実行します。
      ubuntu@ip-172-31-23-251:~$export LANGUAGE=ja_JP.UTF-8
    • 次のコマンドを実行して、「ja_JP.UTF-8」と表示されることを確認します。
      ubuntu@ip-172-31-23-251:~$echo $LANGUAGE
      ja_JP.UTF-8
    • 次のコマンドを実行します。
      ubuntu@ip-172-31-23-251:~$export LANG=ja_JP.UTF-8
    • 次のコマンドを実行して、「ja_JP.UTF-8」と表示されることを確認します。
      ubuntu@ip-172-31-23-251:~$echo $LANG
      ja_JP.UTF-8
    • 次のコマンドを実行します。
      ubuntu@ip-172-31-23-251:~$export LC_ALL=ja_JP.UTF-8
    • 次のコマンドを実行して、「ja_JP.UTF-8」と表示されることを確認します。
      ubuntu@ip-172-31-23-251:~$echo $LC_ALL
      ja_JP.UTF-8
    • 次のコマンドを入力して、「.bashrc」ファイルをテキストエディタviで編集します。
      ubuntu@ip-172-31-23-251:~$vi .bashrc
      ファイルの末尾に次の３行を追加します。
      export LANGUAGE=ja_JP.UTF-8
      export LANG=ja_JP.UTF-8
      export LC_ALL=ja_JP.UTF-8
      変更内容を保存して、viを終了します。
    • 次のコマンドを実行します。
      ubuntu@ip-172-31-23-251:~$sudo locale-gen ja_JP.UTF-8
    • 1度ログアウトして、再度ログインします
      ubuntu@ip-172-31-23-251:~$logout
  9. 次のコマンドを入力します。
    ubuntu@ip-172-31-23-251:~$ sudo apt-get -y install unzip make gcc apache2

JDBCの設定

Payaraの管理画面で、MySQLデータベースに接続するためのJDBCの設定を行います。

  1. 上記の手順で、Payaraコンソール画面を表示します。
  2. 管理画面の左側のツリーから、「Resources」-「JDBC」-「JDBC Connection Pools」を選択します。
    「JDBC Connection Pools」画面で「New」ボタンを押します。
  3. 「New JDBC Connection Pool (Step 1 of 2)」画面で、次のように入力・選択します。
    名前 値
    Pool Name YuimarlPool
    Resource Type javax.sql.DataSource
    Database Driver Vendor MySQL


    「Next」ボタンを押します。
  4. 「New JDBC Connection Pool (Step 2 of 2)」画面の「Additional Properties」で、以下の値を設定します。（下記以外は変更しない）
    名前 値
    DatabaseName yuimarl
    Password yuimarlpass
    URL jdbc:mysql://localhost:3306/yuimarl?characterEncoding=utf8
    Url jdbc:mysql://localhost:3306/yuimarl?characterEncoding=utf8
    ServerName localhost
    User yuimarluser


    「Finish」ボタンを押します。
  5. 管理画面の左側のツリーから、「Resources」-「JDBC」-「JDBC Resources」を選択します。
    「JDBC Resources」画面で「New」ボタンを押します。
  6. 「新規JDBCリソース」画面で、次のように入力・選択します。
    名前 値
    JNDI名 jdbc/yuimarl
    プール名 YuimarlPool

    「OK」ボタンを押します。

Payara自動起動の設定

OSが再起動された時に、Payaraが自動的に起動するように設定を行います。

  1. テキストエディタviで、「payara」という自動起動スクリプトを作成します。
    ubuntu@ip-172-31-23-251:~$sudo vi /etc/init.d/payara
    次のように編集して保存します。 
    \n#!/bin/sh
### BEGIN INIT INFO
# Provides:          payara
# Required-Start:    $local_fs $remote_fs $network $syslog
# Required-Stop:     $local_fs $remote_fs $network $syslog
# Default-Start:     2 3 4 5
# Default-Stop:      S 0 1 6
# Short-Description: StartUP and Shutdown for Payara.
# Description: Written by Masahiro Nitta
### END INIT INFO
. /lib/lsb/init-functions
export LANG=ja_JP.UTF-8
export LC_ALL=ja_JP.UTF-8
export JAVA_HOME=/opt/jdk1.8.0_102
export PATH=$PATH:$JAVA_HOME/bin
export CLASSPATH=.:$JAVA_HOME/jre/lib:$JAVA_HOME/lib:$JAVA_HOME/lib/tools.jar

case "$1" in
start)
/opt/payara41/bin/asadmin start-domain
;;
restart|reload|force-reload)
/opt/payara41/bin/asadmin restart-domain
;;
stop)
/opt/payara41/bin/asadmin stop-domain
;;
*)
;;
esac
  2. 次のコマンドを実行します。
    ubuntu@ip-172-31-23-251:~$sudo chmod +x /etc/init.d/payara
    ubuntu@ip-172-31-23-251:~$ sudo update-rc.d payara defaults 92 08
  3. 自動起動の確認を行うために、次のように再起動を行います。
    ubuntu@ip-172-31-23-251:~$sudo shutdown -r now
    しばらくしてからYuimarlにアクセスできれば設定が正常に行われました。
