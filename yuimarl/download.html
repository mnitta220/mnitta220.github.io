ダウンロード

Yuimarl の MavenプロジェクトのGitリポジトリは、次のGitHubで公開されています。

https://github.com/MasahiroNitta/yuimarl

Gitをインストールして、次のコマンドを実行することによって、YuimarlのGitリポジトリのクローンを取得することができます。

git clone https://github.com/MasahiroNitta/yuimarl.git

※ Gitがインストールされていない環境では、上記Webページの「Clone or download」ボタンから、「Download ZIP」を選択すると、ZIPファイルをダウンロードすることができます。

    • yuimarl-lib プロジェクトのビルド時に、「注釈処理で捕捉されない例外がスローされました。」というエラーが発生する不具合を修正しました。
  • Version 140822
    • プロジェクトの構成を変更しました。
      〔旧バージョン〕
      • yuimarl-common : EJBとライブラリ用のプロジェクト。
      • yuimarl-web : Web画面用のプロジェクト。
      • 〔新バージョン〕
      • yuimarl-lib : 共通ライブラリ用のプロジェクト。
      • yuimarl-ejb : EJB用のプロジェクト。
      • yuimarl-web : Web画面用のプロジェクト。
      • yuimarl-ear : エンタープライズアプリケーション用のプロジェクト。
      EJB(エンタープライズ JavaBeans) と Web の結合を弱めました（疎結合）。旧バージョンでは、WebプロジェクトのマネージドBean（画面の制御を行うクラス）から、直接 EJB を実行していました。これに対し、新バージョンでは、マネージドBean は、EJB のインターフェースを使用するようにしました。
      たとえば、Party の検索・更新などを行う機能は、次のように分けました。
      • PartyService : 検索・更新などの関数を定義したインターフェース
      • PartyServiceBean : 上記の関数を実装したクラス
      マネージドBeanでは PartyService インターフェースを使用します。（実行時には、PartyServiceBean のインスタンスが自動的にインジェクション(注入)されます。）
      これにより、EJB と Web の開発担当者を分けたり、EJB と Web を別サーバーに配置することなどが可能となります。
      また、エンタープライズアプリケーションに変更したことにより、開発環境で実行するのは yuimarl-ear プロジェクトになり、サーバーにデプロイ（配置）するのは、yuimarl-ear 配下の EARファイルとなりました。
    • EJBの単体テスト用プロジェクトを追加しました。
      • yuimarl-test-javadb : Java DB 用の単体テストプロジェクト。
      • yuimarl-test-mysql : MySQL 用の単体テストプロジェクト。
      これは、開発時に EJB の JUnit 単体テストを実行するためのプロジェクトです。Web画面を使わなくても単体テストができますので、開発効率が上がりますし、EJB と Web の分担にもつながります。
    • 数箇所の不具合を修正しました。
  • Version 140819
    • Internet Explorer で正常に動作しなかった不具合を修正しました。

      • yuimarl-db-javadb-export

  • Version 140812
    • スマートフォン向けのページを追加しました。ログイン画面の「デバイス」で「Mobile」を選択してログインすると、スマートフォン向けの画面が表示されます。現在は、人/組織/物の検索・表示機能のみです。
    • yuimarl-commonプロジェクトの jp.yuimarl.util.YuimarlUtil クラスに、writeLog() メソッドを追加しました。このメソッドは、ログファイルにメッセージを出力するために使用します。出力されるログファイルは、glassfish/domains/domain1/logs/server.log です。
    • yuimarl-webプロジェクトの構成を変更しました。mainフォルダを追加し、index.xhtmlファイル等をこのフォルダに入れました。これによって、前バージョンで使用していたredirect.xhtmlファイルが不要となりました。
      また、前バージョンで使用していたcontractsフォルダを使用しないように変更しました。
    • 若干の不具合修正を行いました。
  • Version 140801

    • 機能としては、人・組織・法人・物品の登録ができます。
